Configuração do aplicativo

Vamos começar criando um novo projeto, instalando dependências necessárias e configurando Um aplicativo de servidor básico.

Novo projeto​

  • Node.js & VSCode
  • .NET 6 & VSCode
  • .NET 6 & VS2022

Crie uma nova pasta para seu projeto, navegue até ela na linha de comando e inicialize uma nova Node.js projeto:

npm init -y

Em seguida, instale todas as dependências Node.js que vamos usar. Neste caso, será [Dotenv] (https://www.npmjs.com/package/dotenv) (utilitário para inicializar variáveis de ambiente a partir de um arquivo), [Express.js] (https://expressjs.com) (framework web), um Express.js middleware para lidar com solicitações de 'multipart/form-data' e, finalmente, os vários pacotes do SDK oficial do APS:

caution

A partir de abril de 2024, o APS SDK para Node.js está em beta, o que significa que as interfaces públicas ainda podem mudar. Para evitar possíveis problemas, recomendamos a instalação das versões específicas desses pacotes, conforme mostrado abaixo.

npm install --save dotenv express express-formidable
npm install --save @aps_sdk/autodesk-sdkmanager@0.0.7-beta.1
npm install --save @aps_sdk/authentication@0.1.0-beta.1
npm install --save @aps_sdk/oss@0.1.0-beta.1
npm install --save @aps_sdk/model-derivative@0.1.0-beta.1

As ''dependências'' no seu arquivo 'package.json' agora devem ter a seguinte aparência: (potencialmente com números de versão ligeiramente diferentes):

// ...
  "dependencies": {
    "@aps_sdk/authentication": "0.1.0-beta.1",
    "@aps_sdk/autodesk-sdkmanager": "0.0.7-beta.1",
    "@aps_sdk/model-derivative": "0.1.0-beta.1",
    "@aps_sdk/oss": "0.1.0-beta.1",
    "dotenv": "^16.4.1",
    "express": "^4.18.2",
    "express-formidable": "^1.2.0"
  },
// ...

Finalmente, vamos criar mais algumas subpastas na pasta do projeto que vamos para precisar mais tarde:

  • 'wwwroot' - é aqui que vamos colocar todos os ativos do lado do cliente (HTML, CSS, JavaScript, imagens, etc.)
  • 'rotas' - é aqui que vamos implementar todos os endpoints do servidor
  • 'serviços' - aqui vamos manter toda a lógica do lado do servidor que pode ser compartilhada por diferentes endpoints

Agora, quando você abre sua pasta de projeto no Visual Studio Code pela primeira vez, a estrutura de pastas deve ser semelhante a isto:

Crie uma nova pasta para seu projeto, navegue até ela na linha de comando e inicialize uma nova ASP.NET Projeto principal:

dotnet new web

Em seguida, precisaremos instalar as dependências. Neste caso, será apenas um par de pacotes do SDK oficial do APS:

dotnet add package Autodesk.SdkManager
dotnet add package Autodesk.Authentication
dotnet add package Autodesk.OSS
dotnet add package Autodesk.ModelDerivative

O arquivo '*.csproj' em seu projeto agora deve ser semelhante a este (possivelmente com números de versão ligeiramente diferentes e configurações adicionais do .NET):

<Project Sdk="Microsoft.NET.Sdk.Web">
  <PropertyGroup>
    <TargetFramework>net6.0</TargetFramework>
  </PropertyGroup>
  <ItemGroup>
    <PackageReference Include="Autodesk.Authentication" Version="1.0.0" />
    <PackageReference Include="Autodesk.ModelDerivative" Version="1.0.0" />
    <PackageReference Include="Autodesk.OSS" Version="1.0.0" />
    <PackageReference Include="Autodesk.SdkManager" Version="1.0.0" />
  </ItemGroup>
</Project>

Finalmente, vamos criar mais algumas subpastas na pasta do projeto que precisaremos mais tarde:

  • 'Controladores' - é aqui que vamos implementar todos os endpoints do servidor
  • 'Modelos' - aqui vamos manter toda a lógica do lado do servidor que pode ser compartilhada por diferentes endpoints
  • 'wwwroot' - é aqui que vamos colocar todos os ativos do lado do cliente (HTML, CSS, JavaScript, imagens, etc.)

Agora, quando você abrir sua pasta de projeto no Visual Studio Code pela primeira vez, você será solicitado para configurar seu projeto para desenvolvimento .NET. Aceite o prompt e o editor criará automaticamente uma subpasta '.vscode' com configurações adicionais específicas do .NET, como a configuração de inicialização. A estrutura de pastas no editor deve ser semelhante a esta:

note

Se a pasta '.vscode' não for criada automaticamente, você poderá criá-la por meio do painel lateral Executar & Depurar

Para criar a pasta '.vscode', clique na ferramenta Executar e Depurar no painel lateral esquerdo > criar um arquivo launch.json > Selecionar . NET5+ & .NET Core.

Create a new project in Visual Studio:

Choose the ASP.NET Core Empty template:

Name the project any way you want:

Choose the .NET 6.0 (Long-term support) framework, and disable HTTPS as we don't need it for now:

Once the project is ready, create the following subfolders in your project folder:

  • Controllers - this is where we're going to implement all the server endpoints
  • Models - here we're going to keep all the server-side logic that can be shared by different endpoints
  • wwwroot - this is where we're going to put all the client side assets (HTML, CSS, JavaScript, images, etc.)

Next we will need to install the dependencies. In this case it will be a couple of modules of the official APS SDK. In the Solution Explorer, right-click on Dependencies, and then click on Manage NuGet Packages...:

In the NuGet Package Manager, switch to the Browse tab, and search for and install the following packages:

  • Autodesk.SdkManager
  • Autodesk.Authentication
  • Autodesk.OSS
  • Autodesk.DataManagement
  • Autodesk.ModelDerivative

Configuração do aplicativo​

Nosso aplicativo precisará de algumas entradas de configuração para funcionar corretamente, por exemplo, as credenciais do nosso aplicativo APS (ID do cliente e segredo) para comunicação com os Serviços da Plataforma Autodesk. Passaremos esses parâmetros para nosso aplicativo usando variáveis de ambiente.

  • Node.js & VSCode
  • .NET 6 & VSCode
  • .NET 6 & VS2022

Crie um arquivo 'config.js' na raiz da pasta do projeto e adicione o seguinte código:

config.js
require('dotenv').config();
let { APS_CLIENT_ID, APS_CLIENT_SECRET, APS_BUCKET, PORT } = process.env;
if (!APS_CLIENT_ID || !APS_CLIENT_SECRET) {
    console.warn('Missing some of the environment variables.');
    process.exit(1);
}
APS_BUCKET = APS_BUCKET || `${APS_CLIENT_ID.toLowerCase()}-basic-app`;
PORT = PORT || 8080;
module.exports = {
    APS_CLIENT_ID,
    APS_CLIENT_SECRET,
    APS_BUCKET,
    PORT
};

A biblioteca 'dotenv' inicializa variáveis de ambiente de um arquivo .env local (se houver). Em seguida, basta ler as variáveis de ambiente de 'process.env' e sair do aplicativo imediatamente se alguma das propriedades necessárias estiver faltando. E se nenhum nome de bucket for fornecido para onde nossos designs devem ser armazenados, geramos um anexando o sufixo '-basic-app' para o nosso ID de cliente do aplicativo.

caution

Observe que o serviço de Gerenciamento de Dados requer que os nomes de bucket sejam globalmente exclusivos, e as tentativas de criar um bucket com um nome já usado falharão com '409 Conflict'. Consulte a documentação para mais detalhes.

Next, let's create the .env file in the project folder, and populate it with the required environment variables (replacing <client-id> and <client-secret> with your APS Client ID and Client Secret):

APS_CLIENT_ID="<client-id>"
APS_CLIENT_SECRET="<client-secret>"
caution

Como o arquivo '.env' contém informações confidenciais, certifique-se de que ele não esteja incluído no git! Isso pode ser garantido adicionando a linha '.env' ao arquivo .gitignore.

Agora, para poder executar e depurar nosso aplicativo do Visual Studio Code, precisamos criar a configuração de inicialização. Use Run > Adicionar Configuration no menu para criar uma nova configuração e quando solicitado Para o ambiente específico, escolha Node.js. Isso criará um novo '.vscode' subpasta em seu projeto com um arquivo 'launch.json' onde você pode definir diferentes configurações de inicialização. Substitua o conteúdo do arquivo pelo seguinte:

.vscode/launch.json
{
    "version": "0.2.0",
    "configurations": [
        {
            "type": "node",
            "request": "launch",
            "name": "Launch Server",
            "runtimeExecutable": "npm",
            "runtimeArgs": [
                "start"
            ],
            "skipFiles": [
                "<node_internals>/**/*.js"
            ]
        }
    ]
}

Estamos definindo uma configuração de lançamento único chamada Launch Server que simplesmente Inicie nosso aplicativo usando o comando 'npm start'.

Crie um arquivo 'Startup.cs' na pasta raiz do seu projeto com o seguinte conteúdo:

Startup.cs
using System;
using Microsoft.AspNetCore.Builder;
using Microsoft.AspNetCore.Hosting;
using Microsoft.Extensions.Configuration;
using Microsoft.Extensions.DependencyInjection;
using Microsoft.Extensions.Hosting;
public class Startup
{
    public Startup(IConfiguration configuration)
    {
        Configuration = configuration;
    }
    public IConfiguration Configuration { get; }
    // This method gets called by the runtime. Use this method to add services to the container.
    public void ConfigureServices(IServiceCollection services)
    {
        services.AddControllers();
        var clientID = Configuration["APS_CLIENT_ID"];
        var clientSecret = Configuration["APS_CLIENT_SECRET"];
        var bucket = Configuration["APS_BUCKET"]; // Optional
        if (string.IsNullOrEmpty(clientID) || string.IsNullOrEmpty(clientSecret))
        {
            throw new ApplicationException("Missing required environment variables APS_CLIENT_ID or APS_CLIENT_SECRET.");
        }
    }
    // This method gets called by the runtime. Use this method to configure the HTTP request pipeline.
    public void Configure(IApplicationBuilder app, IWebHostEnvironment env)
    {
        if (env.IsDevelopment())
        {
            app.UseDeveloperExceptionPage();
        }
        app.UseDefaultFiles();
        app.UseStaticFiles();
        app.UseRouting();
        app.UseEndpoints(endpoints =>
        {
            endpoints.MapControllers();
        });
    }
}

A classe 'Startup' é responsável por configurar nosso servidor e seu "middleware", por exemplo, serviço de arquivos estáticos. Também tentamos recuperar o ID e o segredo do cliente do aplicativo APS de variáveis de ambiente (ou de outros provedores de configuração) para uso posterior.

Now, to pass actual configuration values to our application for debugging purposes, we will modify (or create) the appsettings.Development.json file in our project folder with the following content (replacing <client-id> and <client-secret> with your APS Client ID and Client Secret):

{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "APS_CLIENT_ID": "<client-id>",
  "APS_CLIENT_SECRET": "<client-secret>"
}
caution

Desde o 'appsettings. Development.json' arquivo contém informações confidenciais, certifique-se de que ele não está não incluído no git! Isso pode ser garantido adicionando o 'appsettings. Development.json' linha para o arquivo .gitignore.

Servidor básico​

Em seguida, criaremos um aplicativo de servidor básico.

  • Node.js & VSCode
  • .NET 6 & VSCode
  • .NET 6 & VS2022

Crie um arquivo 'server.js' na raiz da pasta do projeto com o seguinte código:

server.js
const express = require('express');
const { PORT } = require('./config.js');
let app = express();
app.use(express.static('wwwroot'));
app.listen(PORT, function () { console.log(`Server listening on port ${PORT}...`); });

Por enquanto, o servidor não está fazendo muito, apenas servindo ativos estáticos da subpasta 'wwwroot' (que está vazio no momento) e relatando quaisquer erros para o console e de volta para o cliente.

Em seguida, vamos adicionar um script '"start": "node server.js'' ao arquivo 'package.json' para que possamos Execute facilmente nosso aplicativo mais tarde:

// ...
"scripts": {
    "start": "node server.js"
}
// ...

Substitua o conteúdo de 'Program.cs' pelo seguinte:

Program.cs
using Microsoft.AspNetCore.Hosting;
using Microsoft.Extensions.Hosting;
public class Program
{
    public static void Main(string[] args)
    {
        CreateHostBuilder(args).Build().Run();
    }
    public static IHostBuilder CreateHostBuilder(string[] args) =>
        Host.CreateDefaultBuilder(args)
            .ConfigureWebHostDefaults(webBuilder =>
            {
                webBuilder.UseStartup<Startup>();
            });
}

Com este código, nosso aplicativo .NET iniciará um servidor ASP.NET com a configuração definido na classe 'Startup'.

E, finalmente, vamos atualizar o arquivo 'launchSettings.json' na pasta 'Propriedades' para fazer certeza de que nosso aplicativo de servidor está usando o protocolo padrão (HTTP) e a porta (8080) que nós será usado ao longo deste tutorial. Atualizar a propriedade 'applicationUrl' do padrão Iniciar perfil para 'http://localhost:8080':

{
  "iisSettings": {
    "windowsAuthentication": false,
    "anonymousAuthentication": true,
    "iisExpress": {
      "applicationUrl": "http://localhost:38524",
      "sslPort": 44323
    }
  },
  "profiles": {
    "my_aps_app": {
      "commandName": "Project",
      "dotnetRunMessages": true,
      "launchBrowser": true,
      "applicationUrl": "http://localhost:8080",
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Development"
      }
    },
    "IIS Express": {
      "commandName": "IISExpress",
      "launchBrowser": true,
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Development"
      }
    }
  }
}
