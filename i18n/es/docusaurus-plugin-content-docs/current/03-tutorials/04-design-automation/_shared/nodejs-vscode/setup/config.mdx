### Instalar paquetes

De forma predeterminada, un proyecto Node.js está vacío, por lo que necesitamos instalar algunos paquetes con npm install. Comencemos con un servidor exprés básico, body-parser para el manejo de JSON, multer para la carga de archivos y, por supuesto, APS.



:::caution

Ejecute una instalación de npm a la vez.


:::



```bash
npm install express --save
npm install express-formidable --save
npm install multer --save
npm install cookie-session --save
npm install forge-apis --save
npm install autodesk.forge.designautomation --save
npm install body-parser --save
npm install form-data --save
npm install socket.io --save
```

> El parámetro --save indica que el módulo debe incluirse en el archivo package.json como una dependencia.

Finalmente abra el package.json y, dentro de "scripts", agregue "start": "nodo start.js", línea. Ahora su carpeta debe tener una carpeta 'node_modules' y su 'package.json' debería verse así

```json title=package.json
{
  "name": "designAutomationSample",
  "version": "1.0.0",
  "description": "",
  "main": "start.js",
  "scripts": {
    "start": "node start.js",
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "MIT",
  "dependencies": {
    "autodesk.forge.designautomation": "^3.0.3",
    "body-parser": "^1.19.0",
    "cookie-session": "^2.0.0",
    "express": "^4.17.1",
    "forge-apis": "^0.9.2",
    "form-data": "^4.0.0",
    "multer": "^1.4.5-lts.1",
    "socket.io": "^4.0.1"
  }
}
```

> El número de versión (por ejemplo, forge-apis 0.9.2) puede variar, era la última versión cuando se creó este tutorial.

### Archivos y carpetas

Para crear una nueva carpeta o archivo, haga clic con el botón derecho en el área "Explorador" a la izquierda y seleccione Nueva carpeta o Nuevo archivo.

Cree una carpeta '/routes/' para todos los archivos del lado del servidor y una carpeta '/wwwroot/' para todos los archivos del lado del cliente.

En este punto, el proyecto debe tener la siguiente estructura

<img src={VsCodeFolderExplorerImg} style={{ width: 300 }} />

Este archivo indica a Visual Studio Code cómo debemos ejecutar nuestro proyecto. Vaya al menú Ejecutar >> Agregar configuración... y, en la ventana Seleccionar entorno que aparece en la parte superior, elija Node.js. En el archivo /.vscode/launch.json que se crea, escriba lo siguiente

```json title=launch.json
{
  // Use IntelliSense to learn about possible attributes.
  // Hover to view descriptions of existing attributes.
  // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Launch Program",
      "program": "${workspaceFolder}/start.js",
      "env": {
        "APS_CLIENT_ID": "your id here",
        "APS_CLIENT_SECRET": "your secret here"
      }
    }
  ]
}
```



:::note


It's important to define ID & Secret as environment variables so our project can later be deployed online. More on this later, in Deployment.


:::
