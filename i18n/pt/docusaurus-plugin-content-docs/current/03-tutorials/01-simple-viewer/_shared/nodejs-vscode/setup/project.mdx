Crie uma nova pasta para seu projeto, navegue até ela na linha de comando e inicialize uma nova
Node.js projeto:

```bash
npm init -y
```

Em seguida, instale todas as dependências Node.js que vamos usar. Neste caso, será
[Dotenv](https://www.npmjs.com/package/dotenv) (utilitário para inicializar variáveis de ambiente a partir de um arquivo),
[Express.js](https://expressjs.com) (framework web), um Express.js [middleware](https://expressjs.com/en/guide/using-middleware.html)
para lidar com solicitações de 'multipart/form-data' e, finalmente, os vários pacotes do SDK oficial do APS:



:::caution

A partir de abril de 2024, o APS SDK para Node.js está em beta, o que significa que as interfaces públicas ainda podem mudar.
Para evitar possíveis problemas, recomendamos a instalação das versões específicas desses pacotes, conforme mostrado abaixo.


:::



```bash
npm install --save dotenv express express-formidable
npm install --save @aps_sdk/autodesk-sdkmanager@0.0.7-beta.1
npm install --save @aps_sdk/authentication@0.1.0-beta.1
npm install --save @aps_sdk/oss@0.1.0-beta.1
npm install --save @aps_sdk/model-derivative@0.1.0-beta.1
```

As ''dependências'' no seu arquivo 'package.json' agora devem ter a seguinte aparência:
(potencialmente com números de versão ligeiramente diferentes):

```json
// ...
  "dependencies": {
    "@aps_sdk/authentication": "0.1.0-beta.1",
    "@aps_sdk/autodesk-sdkmanager": "0.0.7-beta.1",
    "@aps_sdk/model-derivative": "0.1.0-beta.1",
    "@aps_sdk/oss": "0.1.0-beta.1",
    "dotenv": "^16.4.1",
    "express": "^4.18.2",
    "express-formidable": "^1.2.0"
  },
// ...
```

Finalmente, vamos criar mais algumas subpastas na pasta do projeto que vamos
para precisar mais tarde:

- 'wwwroot' - é aqui que vamos colocar todos os ativos do lado do cliente (HTML, CSS, JavaScript, imagens, etc.)
- 'rotas' - é aqui que vamos implementar todos os endpoints do servidor
- 'serviços' - aqui vamos manter toda a lógica do lado do servidor que pode ser compartilhada por diferentes endpoints
