# Painel de controle

## Introdução

Neste tutorial, vamos melhorar o [Simple Viewer](../simple-viewer)
com funcionalidade semelhante a um painel, por exemplo, recuperando e agregando informações
a partir de nossos metadados de design, exibindo os resultados em gráficos e grades de dados e controlando
o estado do espectador com base na interação com esses novos visuais.

All the functionality in this tutorial is implemented as viewer extensions, and will only use
the viewer UI, so you're free to use any APS application you want as your starting point.
Note however that the names of folders, and the way you import JavaScript files into your
web application, might be a little bit different.



:::caution

Algumas partes do código deste tutorial pressupõem que seus elementos de design contêm propriedades específicas
como _Volume_, _Level_, ou _Price_. Se você tem um design específico com o qual deseja trabalhar, você pode
é claro modificar o código para usar outras propriedades, caso contrário, recomendamos usar um dos oficiais
[Arquivos de projeto de exemplo do Revit 2022] (https://knowledge.autodesk.com/support/revit/getting-started/caas/CloudHelp/cloudhelp/2022/ENU/Revit-GetStarted/files/GUID-61EF2F22-3A1F-4317-B925-1E85F138BE88-htm.html)
onde essas propriedades estão sempre presentes.


:::

Usaremos o [Viewer](https://aps.autodesk.com/en/docs/viewer/v7/developers_guide/overview)
API e seus vários recursos, tais como:

- Criação e carregamento de extensões de visualizador
- Consulta de metadados de modelos carregados
- Reagir a eventos do espectador
- Controlar o estado do espectador
- Personalização da barra de ferramentas do visualizador
- Criação de elementos de interface do usuário personalizados

:::
