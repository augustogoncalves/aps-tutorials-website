# Ověřování

V tomto kroku rozšíříme implementaci serveru tak, aby se mohl sám ověřit
na platformu APS a generovat přístupové tokeny pro různé případy použití.



:::tip

Osvědčeným postupem je vygenerovat "interní" token s více možnostmi (například
umožňující vytvářet nebo mazat soubory ve službě Správa dat), které budou použity pouze
serverem a "veřejný" token s menším počtem funkcí, se kterými lze bezpečně sdílet
logika na straně klienta.


:::
