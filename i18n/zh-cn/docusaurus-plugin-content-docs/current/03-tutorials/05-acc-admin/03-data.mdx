## 试一试

像往常一样从 Visual Studio Code 启动（或重启）应用，然后导航到
[http://localhost:8080/api/hubs](http://localhost:8080/api/hubs） 在浏览器中。
服务器应使用您有权访问的所有中心的 JSON 列表进行响应。
尝试复制其中一个集线器的 ID，并将其用于另一个地址：http://localhost:8080/api/hubs/your-hub-id/projects。
在这种情况下，服务器应用程序应使用所有项目的 JSON 列表进行响应
在指定的中心下可用。



:::info

如果跳过了上一步中的登录过程，或重新启动了服务器应用程序，
您可能需要转到 [http://localhost:8080/api/auth/login](http://localhost:8080/api/auth/login）
再次确保在测试之前，所有身份验证数据都在 cookie 中可用
“/api/hubs”终结点。


:::





:::tip

如果您使用的是 Google Chrome，请考虑安装 [JSON 格式化程序](https://chrome.google.com/webstore/detail/json-formatter/bcjindcccaagfpapjjmafapmmgkkhgoa?hl=en）
或类似的扩展程序来自动格式化 JSON 响应。
