프로젝트에 대한 새 폴더를 만들고, 명령줄에서 해당 폴더로 이동하고, 새
Node.js 프로젝트:

```bash
npm init -y
```

다음으로, 사용할 모든 Node.js 종속성을 설치합니다. 이 경우
[도텐브](https://www.npmjs.com/package/dotenv) (파일에서 환경 변수를 초기화하는 유틸리티),
[Express.js](https://expressjs.com) (웹 프레임워크), Express.js [미들웨어](https://expressjs.com/en/guide/using-middleware.html)
'multipart / form-data'요청을 처리하고 마지막으로 공식 APS SDK의 다양한 패키지를 처리합니다.



:::caution

2024년 4월 현재 APS SDK for Node.js는 베타 버전이므로 공용 인터페이스는 여전히 변경될 수 있습니다.
잠재적인 문제를 방지하려면 아래와 같이 이러한 패키지의 특정 버전을 설치하는 것이 좋습니다.


:::



```bash
npm install --save dotenv express express-formidable
npm install --save @aps_sdk/autodesk-sdkmanager@0.0.7-beta.1
npm install --save @aps_sdk/authentication@0.1.0-beta.1
npm install --save @aps_sdk/oss@0.1.0-beta.1
npm install --save @aps_sdk/model-derivative@0.1.0-beta.1
```

'package.json' 파일의 ''dependencies''는 이제 다음과 같아야 합니다.
(잠재적으로 약간 다른 버전 번호 사용) :

```json
// ...
  "dependencies": {
    "@aps_sdk/authentication": "0.1.0-beta.1",
    "@aps_sdk/autodesk-sdkmanager": "0.0.7-beta.1",
    "@aps_sdk/model-derivative": "0.1.0-beta.1",
    "@aps_sdk/oss": "0.1.0-beta.1",
    "dotenv": "^16.4.1",
    "express": "^4.18.2",
    "express-formidable": "^1.2.0"
  },
// ...
```

마지막으로, 프로젝트 폴더에 몇 개의 하위 폴더를 더 만들어 보겠습니다
나중에 필요로하는 것 :

- 'wwwroot' - 모든 클라이언트 측 자산(HTML, CSS, JavaScript, 이미지 등)을 넣을 위치입니다.
- 'routes' - 모든 서버 엔드포인트를 구현할 위치입니다.
- 'services' - 여기서는 서로 다른 엔드포인트에서 공유할 수 있는 모든 서버 쪽 논리를 유지합니다.
