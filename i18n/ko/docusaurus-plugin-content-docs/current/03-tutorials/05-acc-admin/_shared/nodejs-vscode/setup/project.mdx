프로젝트에 대한 새 폴더를 만들고 명령 줄에서 해당 폴더로 이동하십시오.
새 Node.js 프로젝트를 초기화합니다.

```bash
npm init -y
```

다음으로, 사용할 모든 Node.js 종속성을 설치합니다. 이 경우 [dotenv](https://www.npmjs.com/package/dotenv)가 됩니다.
(파일에서 환경 변수를 초기화하는 유틸리티), [Express.js](https://expressjs.com) (웹 프레임워크),
쿠키 기반 세션을 처리하기 위한 [cookie-session](https://www.npmjs.com/package/cookie-session) 및 마지막으로 다음을 포함한 필요한 APS SDK :
- [@aps_sdk/autodesk-sdkmanager](https://www.npmjs.com/package/@aps_sdk/autodesk-sdkmanager),
- [@aps_sdk/인증](https://www.npmjs.com/package/@aps_sdk/authentication),
- [@aps_sdk/데이터 관리](https://www.npmjs.com/package/@aps_sdk/data-management),
- [@aps_sdk/계정 관리자](https://www.npmjs.com/package/@aps_sdk/account-admin):




:::caution

2024년 4월 현재 APS Node.js용 SDK는 베타 버전이며, 이는 공개 
인터페이스는 여전히 변경될 수 있습니다. 잠재적인 문제를 방지하려면 설치하는 것이 좋습니다. 
이러한 패키지의 특정 버전은 아래와 같습니다.


:::



```bash
npm install --save dotenv express cookie-session 
npm install --save @aps_sdk/autodesk-sdkmanager@0.0.7-beta.1 
npm install --save @aps_sdk/authentication@0.1.0-beta.1 
npm install --save @aps_sdk/data-management@0.1.0-beta.1 
npm install --save @aps_sdk/account-admin@0.1.0-beta.1
```

'package.json' 파일의 ''dependencies''는 이제 다음과 같아야 합니다.
(잠재적으로 약간 다른 버전 번호 사용) :

```json
// ...
  "dependencies": {
    "@aps_sdk/authentication": "^0.1.0-beta.1",
    "@aps_sdk/autodesk-sdkmanager": "^0.0.7-beta.1",
    "@aps_sdk/data-management": "^0.1.0-beta.1",
    "@aps_sdk/account-admin": "^0.1.0-beta.1",
    "cookie-session": "^2.1.0",
    "dotenv": "^16.4.5",
    "express": "^4.19.2"
  },
// ...
```

마지막으로, 프로젝트 폴더에 나중에 필요한 하위 폴더를 몇 개 더 만들어 보겠습니다.

- 'wwwroot' - 모든 클라이언트 측 자산(HTML, CSS, JavaScript, 이미지 등)을 넣을 위치입니다.
- 'routes' - 모든 서버 엔드포인트를 구현할 위치입니다.
- 'services' - 여기서는 서로 다른 엔드포인트에서 공유할 수 있는 모든 서버 쪽 논리를 유지합니다.
