### 패키지 설치

기본적으로 Node.js 프로젝트는 비어 있으므로 npm install을 사용하여 몇 가지 패키지를 설치해야 합니다. 기본 익스프레스 서버, JSON 처리를 위한 본문 파서, 파일 업로드를 위한 multer, 그리고 물론 APS부터 시작하겠습니다.



:::caution

한 번에 하나의 npm install을 실행합니다.


:::



```bash
npm install express --save
npm install express-formidable --save
npm install multer --save
npm install cookie-session --save
npm install forge-apis --save
npm install autodesk.forge.designautomation --save
npm install body-parser --save
npm install form-data --save
npm install socket.io --save
```

> --save 매개 변수는 모듈이 package.json 파일에 종속성으로 포함되어야 함을 나타냅니다.

마지막으로 package.json 열고 "스크립트"안에 "start": "node start.js", 줄을 추가하십시오. 이제 폴더에 'node_modules' 폴더가 있어야 하고 'package.json'는 다음과 같아야 합니다.

```json title=package.json
{
  "name": "designAutomationSample",
  "version": "1.0.0",
  "description": "",
  "main": "start.js",
  "scripts": {
    "start": "node start.js",
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "MIT",
  "dependencies": {
    "autodesk.forge.designautomation": "^3.0.3",
    "body-parser": "^1.19.0",
    "cookie-session": "^2.0.0",
    "express": "^4.17.1",
    "forge-apis": "^0.9.2",
    "form-data": "^4.0.0",
    "multer": "^1.4.5-lts.1",
    "socket.io": "^4.0.1"
  }
}
```

> 버전 번호(예: forge-apis 0.9.2)는 다를 수 있으며 이 자습서를 만들 때 최신 버전이었습니다.

### 파일 및 폴더

새 폴더 또는 파일을 만들려면 왼쪽의 "탐색기" 영역을 마우스 오른쪽 버튼으로 클릭하고 새 폴더 또는 새 파일을 선택합니다.

모든 서버 측 파일에 대해 '/routes/' 폴더를 만들고 모든 클라이언트 측 파일에 대해 '/wwwroot/' 폴더를 만듭니다.

이 시점에서 프로젝트의 구조는 다음과 같아야 합니다

<img src={VsCodeFolderExplorerImg} style={{ width: 300 }} />

이 파일은 Visual Studio Code에 프로젝트를 실행하는 방법을 나타냅니다. 메뉴로 이동 실행 >> 구성 추가... 을 클릭하고 맨 위에 표시되는 환경 선택 창에서 Node.js를 선택합니다. 생성된 /.vscode/launch.json 파일에서 다음을 입력합니다

```json title=launch.json
{
  // Use IntelliSense to learn about possible attributes.
  // Hover to view descriptions of existing attributes.
  // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Launch Program",
      "program": "${workspaceFolder}/start.js",
      "env": {
        "APS_CLIENT_ID": "your id here",
        "APS_CLIENT_SECRET": "your secret here"
      }
    }
  ]
}
```



:::note


It's important to define ID & Secret as environment variables so our project can later be deployed online. More on this later, in Deployment.


:::
